Лабораторная работа №3: Pong \(Вариант 13 – Рон\)

Классическая аркадная игра Pong, реализованная на Python с использованием библиотеки Pygame\. Игрок управляет левой ракеткой, а правой управляет компьютерный бот \(или второй игрок по сети\)\. Проект выполнен в рамках курса «Проектирование программного обеспечения интеллектуальных систем» \(часть 2\)\. Вариант 13\. Рон подразумевает игру с ботом, а также усложнение – сетевую игру на двоих\.

Функциональные требования \(из задания\)

1\. Событийно\-ориентированное программирование \(библиотека Pygame\)

2\. Конфигурация во внешних файлах \(JSON\)

3\. Меню: «начать игру», «таблица рекордов», «справка», «выход»

4\. Звуки и фоновая музыка

5\. Диалог при рекорде с вводом имени

6\. Анимации \(след мяча, эффект удара\)

7\. Документация Markdown

8\. Исходники на GitHub

Дополнительно \(усложнение\): Онлайн\-игра на двоих – реализована через TCP\-сервер и клиент\.

Управление

W / ↑ – Движение ракетки вверх

S / ↓ – Движение ракетки вниз

Enter – Выбор пункта меню / Подтверждение

Esc – Выход из текущего состояния / отмена

Backspace – Удаление символа \(при вводе имени\)

Установка и запуск

Клонирование репозитория:

git clone https://github\.com/yourusername/pong\-lab3\.git

cd pong\-lab3

Создание виртуального окружения \(рекомендуется\):

python \-m venv \.venv

source \.venv/bin/activate   \# Linux/Mac

\# или

\.venv\\Scripts\\activate      \# Windows

Установка зависимостей:

pip install \-r requirements\.txt

Запуск игры:

python main\.py

Подробности реализации

Главный игровой цикл и состояния

Игра построена на конечном автомате состояний:

\- MENU – главное меню

\- PLAYING – одиночная игра с ботом

\- HELP – справка

\- LEADERBOARD – таблица рекордов

\- GAME\_OVER – завершение матча

\- NAME\_INPUT – ввод имени при рекорде

\- ONLINE\_CONNECT – ожидание подключения к серверу

\- ONLINE\_PLAYING – сетевая игра

Переключение состояний осуществляется через обработку событий клавиатуры и таймеров\.

Игровой процесс \(одиночный режим\)

Мяч стартует из центра с случайным направлением\. Левая ракетка управляется игроком \(клавиши W/S или стрелки\)\. Правая ракетка управляется ботом, который следит за мячом только когда тот летит в его сторону \(чтобы избежать «читерства»\)\. При касании мяча о верхнюю/нижнюю стенку – отскок\. При выходе за левую границу – гол боту \(счёт \+1 боту\), за правую – гол игроку\. Матч идёт до 5 очков \(настраивается в config/config\.json\)\. После гола мяч сбрасывается в центр и летит в сторону проигравшего \(для честности\)\.

Управление ботом

Класс BotController \(game/bot\.py\) реализует простую логику:

if ball\.vx > 0:  \# только если мяч летит в сторону бота

    if paddle\.centery < ball\.centery \- tolerance:

        paddle\.move\_down\(\)

    elif paddle\.centery > ball\.centery \+ tolerance:

        paddle\.move\_up\(\)

Параметр tolerance \(задаётся в конфиге\) определяет «зону бездействия» бота, что делает игру более естественной\.

Анимации

След мяча: массив последних позиций мяча \(trail\_points\)\. Каждый кадр добавляется текущий центр, а старые точки отрисовываются с уменьшающимся размером и прозрачностью \(имитация движения\)\. Эффект удара: при столкновении мяча с ракеткой или стенкой вокруг мяча появляется расширяющийся жёлтый круг, который исчезает за долю секунды \(hit\_animation\_timer\)\. Анимация гола: в центре экрана отображается крупная надпись «GOAL\!» с затуханием \(альфа\-канал\)\. Все анимации реализованы через обновление таймеров в игровом цикле без использования дополнительных потоков\.

Звук и музыка

Класс SoundManager \(game/sound\_manager\.py\) инициализирует звуковую подсистему Pygame и загружает файлы из assets/\. При ударе мяча о ракетку или стенку проигрывается hit\.wav\. При голе – goal\.wav\. Фоновая музыка \(background\_soft\_loop\.wav\) запускается в бесконечном цикле с громкостью 0\.25 \(настраивается в config/config\.json\)\. Если звуковые файлы отсутствуют, игра продолжает работу без звука \(ошибки перехватываются\)\.

Таблица рекордов

Рекорды хранятся в data/leaderboard\.json в виде списка объектов \{"name": "\.\.\.", "score": \.\.\.\}\. При завершении матча, если счёт игрока строго больше текущего рекорда \(первой записи\), открывается диалог ввода имени\. После ввода имени запись добавляется, список сортируется по убыванию, обрезается до 10 записей и сохраняется\. Таблица рекордов отображается в соответствующем пункте меню\.

Сетевая игра \(усложнение\)

Реализована одновременная игра по локальной сети \(TCP\)\. Сервер \(network/server\.py\) запускается отдельно и ожидает подключения двух клиентов\. При подключении каждому клиенту назначается роль \(left или right\)\. Сервер выполняет все игровые вычисления \(физику, столкновения, счёт\) и рассылает состояние игры обоим клиентам с частотой 60 Гц\. Клиенты отправляют только команды управления \(up/down/stop\)\. Протокол обмена – JSON с разделителем \\n \(построчная передача\)\. Если один из клиентов отключается, игра останавливается, и сервер переходит в состояние waiting\.

Запуск сетевой игры:

1\. Запустить сервер: python \-m network\.server \(по умолчанию слушает на 0\.0\.0\.0:5000\)\.

2\. Запустить игру python main\.py и в главном меню выбрать «Сетевая игра»\. Клиент автоматически подключится к локальному серверу\.

3\. Повторить для второго игрока \(на том же или другом компьютере, указав IP\-адрес сервера\)\.

Правила игры \(из справки\)

Игра ведётся до 5 очков \(настраивается\)\. Очко засчитывается, когда мяч пересекает левую или правую границу поля\. Левая ракетка – игрок \(или первый клиент\), правая – бот \(или второй клиент\)\. Бот следит за мячом только тогда, когда мяч летит в его сторону\. В сетевом режиме оба игрока управляют своими ракетками, сервер синхронизирует состояние\.
